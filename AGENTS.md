# AGENTS.md

## Что это за проект

`Alesia AI` — demo-приложение с 4 основными подсистемами:

- `avatar` — 3D-сцена и аватар
- `voice` — голосовые рантаймы Gemini / Yandex
- `browser` — открытие, просмотр и действия по сайтам
- `knowledge` — локальные источники знаний

## Главная цель проекта

Сделать демо, которое легко запускать, быстро править и безопасно сопровождать без лишних догадок и без раздувания контекста.

## Ключевые папки

- `project-docs/` — короткие рабочие документы по проекту
- `demo-content/` — контент демо: конфиг, промпты, приветствия
- `src/` — клиент
- `server/` — сервер, маршруты, websocket, browser runtime
- `test/` — архитектурные, серверные, клиентские, e2e и live-проверки
- `runtime-data/` — runtime-файлы и логи

## С чего начинать любую задачу

Сначала прочитать:

1. `project-docs/00_INDEX.md`
2. `project-docs/02_CURRENT_STATE.md`

Потом открывать только те файлы кода, которые реально относятся к задаче.

## Как запускать проект

- `npm install`
- `npm run dev` — клиент и сервер вместе
- `npm run dev:server` — только сервер
- `npm run dev:client` — только клиент
- `npm run start` — запуск сервера без watch

## Как запускать проверки

- `npm run lint`
- `npm test`
- `npm run test:architecture`
- `npm run build`
- `npm run test:e2e`
- `npm run verify`
- `npm run test:live` — требует секреты
- `npm run demo:gate`

## Правила работы агента

- Общайся с пользователем на русском языке.
- Пиши коротко, простым человеческим языком, без лишних сложных терминов.
- Работай только в рамках текущей задачи.
- Не пытайся чинить все подряд.
- Не трогай незапрошенные части проекта.
- Не делай рефакторинг, смену стиля или архитектуры без явного запроса.
- Не меняй интерфейс без четкого согласования с пользователем.
- Не удаляй файлы без необходимости.
- Если задача локальная, не сканируй весь проект без необходимости.
- Сначала читай docs, потом код.
- Не тащи в контекст все подряд. Держи в контексте только то, что полезно для текущей задачи.
- При неясности сначала задай вопрос, а не гадай.
- Если задача касается внешнего API, версии SDK или документации, сначала проверь актуальное состояние по официальным источникам.
- Если уместно использовать инструмент или скрипт, используй его.
- Если задача повторяется, предложи упаковать ее в отдельный инструмент или скрипт.
- При программировании избегай монолитов. Держи модули небольшими и с одной понятной ответственностью.
- После заметных изменений обновляй только затронутую документацию:
  - `project-docs/02_CURRENT_STATE.md`
  - `project-docs/03_NEXT_STEPS.md`
  - `project-docs/04_DECISIONS.md` — если решение реально изменилось
- Не плодить лишние файлы без причины.
- Для Google/Gemini запрещено добавлять или использовать модели ниже 3.1. Live-рантайм должен использовать `models/gemini-3.1-flash-live-preview`, TTS — `gemini-3.1-flash-tts-preview`.

## Что важно помнить

- Сейчас опорная цель — аккуратная работа с проектом, а не починка всех старых проблем за один заход.
- Если видишь побочную проблему, зафиксируй ее кратко в docs или в ответе пользователю, но не исправляй без запроса.
